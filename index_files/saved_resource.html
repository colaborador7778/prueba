                                                                                                                    Start Bootstrap Mod David Terwilliger a year ago

                                                                                                                    Unless the screen size is below 768px, the navigation will not scroll down with the page. On large screens, when you scroll down, the navigation is not fixed, but it will appear right when you start scrolling up. The idea behind this interaction is to make the reading space as clean and minimal as possible, but when a user wants to find the navigation bar once again it takes their normal behavior (scrolling up) and just gives them what they want instantly (rather than having to scroll to the top of the page).

                                                                                                                    To change this you can remove the JavaScript in the clean-blog.js file that is causing the menu bar to act this way, and you can set the menu to something like navbar-fixed-top and add 50px of padding to the top of the body element.

                                                                                                                            Start Bootstrap Mod Juanma Navarro a year ago

                                                                                                                            It's a UX decision really. On a blog, like this one, it's nice to have a 'distraction free' reading experience. As the user keep scrolling down and reading the article, they wont be disturbed by the menu bar. The second they feel like they need to go back, the reaction will be to scroll to the top of the page because the menu bar isn't there. However, when they start scrolling the menu bar conveniently appears - almost like it knows what they want!
